Build a mobile-first responsive web app called “InnerMeter”.

==================================================
PROJECT GOAL
==================================================

Create a Korean-first personality test web app MVP with a clean, modern, mobile-friendly UI.
This is NOT a fixed mobile-width app. It must be a responsive web app optimized for mobile first, but also look good on tablet and desktop.

The service concept:
- Personality tests
- Relationship tests
- Fun quizzes
- Simple tarot / fortune content

The initial language is Korean only.
However, the project structure must be prepared so that multilingual support can be added later.
Do NOT hardcode all UI text directly inside components if possible.
Separate reusable content/data where reasonable.

Brand:
- Name: InnerMeter
- Tagline: Measure Your Inner Self

==================================================
TECH STACK
==================================================

Use:
- React
- Next.js (App Router if available)
- TypeScript
- Tailwind CSS

Requirements:
- Clean folder structure
- Reusable components
- Good readability
- Fast loading
- SEO-friendly page metadata where possible

==================================================
DESIGN / UI STYLE
==================================================

Style direction:
- Clean
- Modern
- Soft but premium
- Minimal clutter
- Friendly, trendy, shareable test-site feeling
- App-like convenience, but clearly a web service

Important UI rules:
- Mobile-first responsive design
- Do NOT lock the screen to one narrow fixed width like a fake phone mockup
- Use centered responsive layout with a comfortable max-width
- Large tap targets
- Clear cards
- Rounded corners
- Soft shadows
- Simple hierarchy
- Plenty of spacing
- Sticky or always-visible progress area when helpful
- Fast and intuitive navigation

Recommended layout behavior:
- Mobile: optimized first
- Tablet/Desktop: gracefully expand with centered content and larger spacing

==================================================
SITE STRUCTURE
==================================================

Create these pages/routes:

1. Home page
- Route: /
- Show brand name and tagline
- Show hero section
- Show trending tests section
- Show category cards
- Show “One Card Tarot” section
- Show recommended tests
- Show CTA to start testing
- Show footer with links

2. Tests list page
- Route: /tests
- Show all tests
- Allow filtering by category:
  - 성격 테스트
  - 연애 테스트
  - 재미 테스트
- Each test should have:
  - title
  - short description
  - category
  - thumbnail/card style preview
  - start button

3. Individual test page
- Route: /tests/[slug]
- One question at a time UI
- Show current progress
- Show answer options as large buttons/cards
- Support previous/next behavior when reasonable
- Save temporary answers in state
- At completion, calculate result and move to result page

4. Result page
- Route: /results/[slug]
- Show result title
- Show one-line summary
- Show detailed explanation
- Show strengths
- Show caution points
- Show “recommended next tests”
- Show share section
- Show retry button
- Show back to tests button

5. Tarot page
- Route: /tarot
- Create a simple “One Card Tarot” experience
- User taps to draw one card
- Randomly show one tarot card result from prepared data
- Show card name, short meaning, love/work/energy message
- Keep this MVP simple and visually appealing

6. About page
- Route: /about
- Short service intro
- Explain that InnerMeter provides fun personality/relationship/self-discovery content

7. Privacy Policy page
- Route: /privacy

8. Terms page
- Route: /terms
